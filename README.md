## Architecture Principles

My backend projects follow **Clean Architecture** with strict layered boundaries:

```text
API Layer          -> Controllers, Middleware, Program.cs
Application Layer  -> Use Cases, DTOs, Service Interfaces
Infrastructure     -> Repositories (EF Core), External Services
Domain Layer       -> Entities, Value Objects, Repository Interfaces
```

The rule is simple: dependencies point inward, and the domain stays independent from external concerns.
